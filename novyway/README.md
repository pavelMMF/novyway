# Novyway («Новый Путь»)

Демонстрационное веб-приложение для публичных проверяемых голосований. Вес участника зависит от подтверждённого уровня знаний в конкретной категории, а голос, снимок весов и административные изменения проектируются как проверяемые данные Aptos.

## Запуск

```bash
pnpm install --frozen-lockfile
pnpm run dev
```

Откройте адрес, который напечатает Vite, обычно `http://localhost:5173`.

Production-сборка:

```bash
pnpm run build
pnpm run preview
```

Самодостаточный HTML без сервера:

```bash
pnpm run build:single
```

Файл будет создан в `dist-single/index.html`.

## Что реализовано

- обзор сети, квалификаций, активных решений, событий и экзаменов;
- публичные дробные бюллетени и история переголосований;
- документы с поправками и голосованием по точному пункту;
- единое пространство документов: список, 3D-граф и совмещённый режим;
- темы документов, фильтры, сортировка, пользовательские связи и граф-пространства;
- профиль с редактируемыми контактами и переходами к экзаменам по категории;
- понятный расчёт веса по зафиксированному снимку;
- админ-панель для квалификаций, политик, голосований, тем и графов;
- светлая, тёмная и системная тема, RU/EN, отдельный язык типов данных;
- 15 экзаменов по пяти категориям и трём уровням: сценарии, одиночный и множественный выбор, числовые ответы;
- PWA, адаптивная навигация, звуки и скрытая «Лаборатория аргументов» со 150 логическими задачами и серверным счётом;
- ссылка на [Telegram-канал](https://t.me/online_council).

## Актуальный серверный режим

Production-сборка запускается через `Start-Sovet-Online.exe`. Локальный сервисный центр владельца инфраструктуры открывается через `Sovet-Online-Admin.exe`; он управляет uptime, PostgreSQL, relayer, резервными копиями и процессом сайта. Отдельная governance-панель `/#/admin` находится внутри сайта и доступна только Aptos-адресам из on-chain списка администраторов Совета. Аккаунты, SIWA-сессии, профили и журнал операций хранятся в PostgreSQL 17 вне репозитория. Aptos Connect предоставляет вход через Google и Apple с автоматическим созданием некастодиального Aptos-адреса.

Старое описание ниже о полностью локальном `localStorage` относится к демонстрационным данным интерфейса. Оно больше не относится к аккаунтам, серверным сессиям и новым транзакциям.

## Важное ограничение

Часть наполнения интерфейса всё ещё демонстрационная. Аккаунты, профиль, сессии, настройки сервиса и новые relayer-операции уже хранятся в PostgreSQL; Aptos остаётся источником истины для голосов и governance-состояния. Формы governance-панели пока создают локальные черновики и не должны считаться опубликованными изменениями до включения подписания административных транзакций после проверки точного ABI контракта.
