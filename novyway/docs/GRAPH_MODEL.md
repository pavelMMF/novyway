# Логика документов и графа

## Как появляются узлы

Граф строится детерминированно из текущего состояния, поэтому одинаковые данные дают одинаковую исходную раскладку. Категории размещаются по кольцу, документы группируются вокруг категории, а связанные сущности идут дальше по цепочке:

`тема/категория → документ → поправка → голосование → снимок`.

Receipts связываются с голосованием, политика категории — с категорией и снимками. Пользовательские связи между документами добавляются поверх этой структуры.

## Что означает форма

| Сущность | Форма | Смысл |
|---|---|---|
| Категория/тема | ромб-октаэдр | смысловой центр кластера |
| Документ | многогранник | самостоятельный проверяемый текст |
| Активное голосование | компактная светящаяся шестигранная шайба | действие требуется сейчас |
| Завершённое голосование | плоский полупрозрачный маркер | историческое решение |
| Поправка, receipt, policy | сфера | событие или зависимая сущность |
| Снимок | куб | зафиксированный набор параметров |

Цвет показывает категорию или группу документа. Активное голосование отличается не только цветом: вокруг связанного документа или поправки вращается тонкая орбита, сам узел даёт мягкое аддитивное свечение, а ведущие к нему рёбра подсвечиваются голубым. Завершённые голосования получают статичный полупрозрачный маркер. Легенда объясняет оба состояния без опоры только на цвет.

## Связь экологии и образования

Если экологический документ ссылается на образовательный, администратор выбирает оба документа и задаёт подпись связи, например «Экологическое образование». В графе появляется отдельное ребро между двумя документами, даже если они находятся в разных цветовых кластерах. При выборе документа инспектор показывает эту связь и позволяет центрировать второй узел.

## Поведение камеры

- обычный клик выбирает узел и открывает инспектор, но не меняет положение камеры;
- «Центрировать» плавно переводит фокус к выбранному узлу;
- выбор связи в инспекторе центрирует связанный узел;
- «Сбросить вид» возвращает исходную камеру;
- боковая/нижняя навигация остаётся доступной, поэтому из графа всегда можно выйти.

## Пользовательские графы

Администратор создаёт граф-пространство, задаёт название и отмечает документы. Пространство появляется в селекторе раздела «Документы и связи». Оно не копирует документы: это сохранённое представление над общим реестром.

Текущий demo-store хранит пространства до перезагрузки. В production нужны ресурсы:

- `graph_spaces(id, organization_id, name, created_by, created_at)`;
- `graph_space_documents(space_id, document_id)`;
- `document_relations(id, from_document_id, to_document_id, relation_type, label, evidence_hash)`.

## Темы документов

У документа одна главная тема и ноль или больше дополнительных. Фильтр темы возвращает документ, если тема совпала с любой из них. Главная тема определяет основной смысловой кластер; дополнительные темы создают пересечения для поиска и будущих графовых связей.
