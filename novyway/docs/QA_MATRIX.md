# Матрица проверки

## Автоматические проверки

- TypeScript: `tsc -b`;
- lint: `oxlint`;
- production build: `vite build`.

## Разрешения

| Профиль | Размер | Что проверяется |
|---|---:|---|
| Desktop | 1280×720 и шире | sidebar, 2/3-column grids, combined documents view |
| iPhone | 390×844 | safe-area, bottom navigation, touch controls, no horizontal overflow |
| Android | 412×915 | cards, filters, graph inspector, forms, no horizontal overflow |
| Small mobile | 360×800 | text wrapping, tables, segmented controls |

## Ключевые сценарии

1. Обзор открывает сеть, экзамены, граф, голосование и расчёт веса.
2. Профиль редактирует имя/почту/Telegram и сохраняет настройки.
3. Квалификация открывает экзамены нужной категории.
4. Документы переключаются между списком, 3D и combined.
5. Фильтр находит главную и второстепенную тему.
6. Граф отображает canvas, орбиты и свечение активных голосований, исторические маркеры и sidebar/bottom-nav.
7. Выбор узла не двигает камеру; центрирование происходит только явной командой.
8. Администратор создаёт тему, назначает темы, связь и граф-пространство.
9. Тёмная тема меняет semantic tokens и остаётся читаемой.
10. На мобильной ширине `scrollWidth <= clientWidth`.
11. Каталог содержит 15 экзаменов; внутри попытки одновременно показано только одно задание.
12. Клик по знаку логотипа открывает «Лабораторию аргументов»: ответ заранее отсутствует в API, попытка меняет серверный счёт, Escape и кнопка закрытия возвращают на текущую страницу.

## Браузеры

Основная ручная проверка проводится в Chromium. Перед production-релизом также нужны Safari iOS и Firefox desktop на реальных/облачных устройствах, особенно для WebGL, WebAudio, PWA install и safe-area.
