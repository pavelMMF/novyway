# Ревью релиза 17 июля 2026 года

> Исторический снимок. Актуальное состояние, включая проверенный source digest и включаемые ограниченные sponsored-транзакции, описано в `START_HERE.md` и `PRODUCTION_SECURITY_AND_MIGRATION.md`.

## Итог

Проект пригоден для закрытой демонстрации на Aptos Testnet. Аккаунты, сессии, профиль, логическая игра и операционные данные работают через PostgreSQL. Чтение опубликованных документов и состояния голосований связано с Testnet. Проект пока нельзя называть полностью производственной системой голосования: sponsored-запись закрыта до воспроизводимой сборки Move-модуля, а документы, экзамены, квалификации и часть административных форм всё ещё используют демонстрационное состояние.

## Что исправлено

- защищены повторная и конкурентная отправка голосов, idempotency key теперь связан с точным содержимым запроса;
- перед fee-payer подписью сохранённый BCS разбирается и сверяется с типизированным intent: сеть, адреса, gas, Move-функция и каждый аргумент;
- неподтверждённая регистрация больше не резервирует email в основной таблице пользователей;
- HTTPS принимает только host-only `HttpOnly`-сессию, CSRF-cookie имеет `SameSite=Strict`, сессия обновляется после смены способа входа;
- production не принимает локальную development-cookie;
- ограничители запросов не доверяют произвольному `X-Forwarded-For`;
- смена email выполняется атомарно, коды хранятся как HMAC и не печатаются в журнал;
- сброс creator-аккаунта закрывается при недоступности Aptos, а не разрешается по ошибке;
- серверные секреты создаются эксклюзивно, ограничиваются ACL и проверяются по длине;
- Google, Apple и внешний Aptos-кошелёк можно активировать в уже открытом профиле без выхода;
- кошелёк-адаптер загружается только на экранах, где он нужен: начальный JavaScript — около 69 КБ gzip вместо 676 КБ;
- автообновление нативной операторской панели больше не перетирает несохранённые настройки;
- добавлены HSTS, запрет cross-domain policy-файлов, CSP, запрет iframe и MIME sniffing.

## Вход и подпись

Кнопки Google и Apple используют Aptos Connect: это не отдельный вход в Google Profile API, а создание или восстановление некастодиального Aptos-аккаунта через провайдера. Email и отдельный пароль после такого входа необязательны. Обычный Aptos-кошелёк остаётся отдельным способом входа. В интерфейсе скрыты внутренние термины `Keyless`, `SIWA` и `AIP-62`.

Платформа может оплатить gas, но не может незаметно подписать голос вместо пользователя. Внешний кошелёк всё равно показывает подтверждение транзакции. Это обязательное свойство некастодиальной модели, а не лишний шаг интерфейса.

## Лаборатория аргументов

- 150 оригинальных задач: 15 семейств ошибок по 10 общественным темам;
- русский и английский варианты;
- ответ и объяснение не отправляются браузеру до попытки;
- вошедшему пользователю засчитывается одна попытка на задачу;
- PostgreSQL хранит общий счёт, число ответов и текущую серию;
- гостю доступен временный счёт без записи в профиль.

## Реальные и демонстрационные данные

| Область | Состояние |
|---|---|
| Аккаунты, профиль, сессии, восстановление | PostgreSQL, серверная реализация |
| Логическая игра и счёт | PostgreSQL, серверная реализация |
| Публичные якоря документов и Testnet-чтение | Aptos Testnet |

## Проверки

```bash
pnpm run lint
pnpm run build
pnpm run verify:sponsored-intent
pnpm run verify:logic
pnpm run verify:logic:storage
pnpm run verify:credentials
pnpm run verify:recovery
pnpm run verify:creator
pnpm run preflight:creator
pnpm audit --prod
```

На момент ревью сборка, lint, 150 задач, конкурентная запись счёта в PostgreSQL, проверки credentials/recovery/creator и аудит production-зависимостей проходят. `preflight:creator` намеренно показывает, что локальный bootstrap ещё не применён: это действие требует приватного пароля владельца в нативном приложении.
