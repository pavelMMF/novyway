# Безопасность, обслуживание и перенос

Актуально для демонстрационного запуска `novyway.com` на домашнем Windows-сервере.

## Два разных контура управления

1. `/#/admin` — управление Советом: квалификации, политики, голосования и администраторы. Доступ вычисляется по Aptos и учётной записи пользователя.
2. `Sovet-Online-Admin.exe` — локальное приложение владельца инфраструктуры. Оно показывает состояние Node.js, PostgreSQL, Aptos и резервных копий. Приложение обращается только к `127.0.0.1:4177` и предъявляет отдельный локальный ключ.

Публичный сайт не раздаёт операторский интерфейс. Ключ оператора, SMTP-пароль, ключи управляемых кошельков и пароль PostgreSQL находятся вне репозитория в `%LOCALAPPDATA%\SovetOnline\secrets` с ограниченными ACL.

## Учётные записи и пароли

- Пароли хешируются `scrypt` с индивидуальной случайной солью. Исходный пароль не сохраняется и не может быть восстановлен из базы.
- Управляемые Aptos-ключи шифруются `AES-256-GCM` отдельным ключом сервера.
- Сессия хранится в PostgreSQL; cookie имеет `HttpOnly`, `SameSite` и `Secure` на публичном домене. Мутации защищены CSRF-токеном и проверкой Origin.
- Восстановление пароля доступно только аккаунтам, у которых уже был вход по почте и паролю. Оно не превращает Google, Apple или внешний кошелёк в парольный аккаунт.
- Для защищённого аккаунта создателя восстановление по одной почте запрещено. Права суперадмина выдаются только сессии, созданной Aptos-подписью, и такая сессия живёт не более 12 часов.
- Код восстановления одноразовый, хранится только как HMAC, действует 10 минут и имеет лимит попыток. После смены пароля все сессии аккаунта удаляются.

## Служебная почта

Скрипт не создаёт почтовый ящик у провайдера. Сначала создайте отдельный адрес, например `security@novyway.com` у почтового провайдера или отдельный Gmail-аккаунт. Включите двухфакторную защиту и выпустите пароль приложения.

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\server\Setup-Mail.ps1
node .\server\Test-Mail.mjs ваш-адрес@example.com
```

SMTP-секрет вводится скрыто и не попадает в Git. После настройки перезапустите сайт.

## Что защищает приложение

- PostgreSQL принимает подключения только на `127.0.0.1:55432`, использует SCRAM и параметризованные SQL-запросы.
- Тело JSON ограничено 32 КБ, а для Aptos-подписи — 64 КБ.
- React не вставляет пользовательский HTML; CSP запрещает сторонние скрипты, объекты и встраивание сайта во фрейм.
- Проверяются Host, Origin, CSRF и типы входных данных через Zod.
- Локальный лимитер учитывает IP и HMAC от адреса почты, регулярно удаляет просроченные записи и ограничен по памяти.
- Режим технических работ действительно блокирует публичные изменяющие запросы.

## Что должен защищать Cloudflare

Cloudflare Tunnel скрывает домашний IP и не требует входящих портов. В Cloudflare Free следует оставить проксирование включённым и добавить правила:

- Managed Challenge для необычного трафика к `/api/auth/*`;
- лимит на `/api/auth/password/login` и `/api/auth/password/reset/*`;
- более высокий лимит на `/api/v1/*`;
- блокировку стран, ASN или источников только при подтверждённой атаке;
- уведомления о всплеске запросов и ошибках 5xx.

Cloudflare принимает на себя объёмный DDoS и сетевых ботов. Приложение всё равно отвечает за лимиты конкретного аккаунта, расходы релейера и авторизацию. Turnstile ещё не включён: до его подключения нельзя считать защиту регистрации от распределённых ботов законченной.

## Спонсированные транзакции

Спонсирование разрешено только в Aptos Testnet и включается оператором. Перед каждой подписью сервер сверяет source digest опубликованного пакета и SHA-256 всех трёх Move-модулей, сам строит транзакцию из типизированного intent и повторно проверяет сеть, sender, fee payer, gas, function ID и каждый аргумент.

Действуют два независимых лимита: на пользователя и общий часовой бюджет. Глобальный лимит фиксируется атомарно в PostgreSQL под advisory lock, поэтому параллельные запросы не могут превысить бюджет. Переменная `SPONSORSHIP_EMERGENCY_LOCK=1` немедленно блокирует новые intent и отправку уже подготовленных транзакций.

Пользовательским кошелькам Testnet APT автоматически не выдаются. После перевода на некастодиальный адрес платформа не может ограничить дальнейшую трату монет. Голос кодируется аргументами `electionId / yesBps / noBps / abstainBps`, а не количеством APT, поэтому fee-payer sponsorship даёт нужный UX без передачи пользователю свободного баланса.

Перед первым публичным голосованием остаются эксплуатационные меры: Turnstile для финансируемых действий, оповещение о низком остатке relayer, дневной бюджет и ручная проверка резервной копии.

## Резервная копия, которую можно перенести

Обычный `pg_dump` недостаточен: без ключа шифрования управляемые кошельки после переноса не расшифруются. Создавайте зашифрованный recovery-контейнер:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\server\Export-Recovery-Bundle.ps1
```

Скрипт делает дамп PostgreSQL, добавляет локальные секреты и манифест с SHA-256, затем шифрует контейнер `AES-256-GCM` ключом, полученным через `scrypt` из отдельной фразы. Незашифрованные временные файлы удаляются. Фразу нельзя хранить на том же ноутбуке.

Проверить контейнер без автоматического восстановления:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\server\Open-Recovery-Bundle.ps1 `
  -Bundle "C:\path\backup.sovetbackup" -Destination "C:\path\inspection"
```

Храните минимум одну копию вне ноутбука и периодически проверяйте `pg_restore --list` и восстановление в отдельную тестовую базу.

## Чистые архивы проекта

```powershell
# Исходники, production-сборка, документация и EXE; без зависимостей, логов, секретов и музыки
powershell -NoProfile -ExecutionPolicy Bypass -File .\server\Export-Portable-Project.ps1

# Частный перенос между своими компьютерами, включая локальный саундтрек
powershell -NoProfile -ExecutionPolicy Bypass -File .\server\Export-Portable-Project.ps1 -IncludePrivateMusic
```

Зависимости восстанавливаются `pnpm install --frozen-lockfile`; PostgreSQL и локальные данные восстанавливаются отдельно. `node_modules` никогда не надо переносить в чат или Git.

## Порядок переноса на новый сервер

1. Остановить изменяющие операции режимом технических работ.
2. Создать и скопировать исходный архив и recovery-контейнер.
3. Установить Node.js и PostgreSQL на новом сервере.
4. Распаковать проект, установить зависимости, собрать `pnpm run build`.
5. Открыть recovery-контейнер, восстановить секреты с ACL и базу через `pg_restore`.
6. Запустить `Start-Sovet-Online.exe`, проверить локальный `/__health`.
7. Перенести Cloudflare Tunnel на новый сервер, не меняя публичный домен.
8. Проверить вход, чтение Aptos, отправку почты и только затем отключить старый сервер.
