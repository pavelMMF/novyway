# Состав Совета и фоновая музыка

## Управление составом

Публичная панель `/admin` отделена от локальной операторской панели сервера. Доступ определяется не ролью PostgreSQL, а фактическим списком `weighted_voting::admins` в Aptos Testnet. Вкладка «Состав Совета» доступна только адресу `weighted_voting::creator`.

Creator может:

- назначить зарегистрированный Aptos-аккаунт напрямую;
- удалить обычного администратора, но не себя;
- допустить или исключить аккаунт из будущих равных голосований;
- создать выборы администратора по экспертным весам темы либо по правилу «один допущенный аккаунт — один голос»;
- исполнить принятый результат отдельной транзакцией.

Пользователи для выбора загружаются из PostgreSQL, но база не выдаёт полномочия. Каждое изменение подписывает подключённый creator-кошелёк и фиксирует Aptos. Равные выборы показываются в реестре голосований и используют существующий relayer для спонсированной комиссии.

Relayer Testnet пополнен на 1 APT и спонсирование включено. Текущий лимит — 20 оплачиваемых голосов на один пользовательский аккаунт в час. Пополнение: `0x5a07a6c5e34d9824620f9a20ccbe1f108c5dc0e467be2fdf858349a7e1663f8b`.

## Музыкальный тракт

Интерфейсные сигналы и музыка имеют отдельные переключатели и громкость. Музыка по умолчанию выключена; выбор пользователя сохраняется на устройстве. После каждой загрузки страницы воспроизведение ждёт 30 секунд, даже если музыка была включена в прошлой сессии. Плеер:

- воспроизводит семь треков по порядку (`01 → 02 → 03 → 04 → 05 → 06 → 07`);
- загружает только текущий трек, следующий начинает готовить перед переходом;
- использует десятисекундные плавное нарастание, затухание и переход между треками;
- начинает с 59% на пользовательском слайдере и дополнительно приглушает громкость на плеере (×0,25 от значения слайдера);
- по умолчанию плавно ставится на паузу, когда вкладка скрыта; в настройках можно разрешить воспроизведение вне активной вкладки;
- позволяет кликнуть по динамику для мгновенного отключения звука, а при наведении или фокусе показывает горизонтальный регулятор громкости;
- не попадает в service-worker cache и single-file сборку;
- получает файлы по HTTP Range (`206 Partial Content`).

Файлы — собственные треки сайта (`Media/music` / `media/music`, регистр не важен на Windows), не лицензированный сторонний материал. Обслуживаются напрямую из `media/music` (см. `musicRoot` в `server/static-server.mjs`), в Git не попадают (`media/music/` в `.gitignore`) просто из-за размера, а не из-за прав.

Публичный показ и сам плейлист управляются прямо в коде, без переменных окружения: константа `PUBLIC_MUSIC_ENABLED` и массив `musicTracks` в начале `server/static-server.mjs` (рядом с комментарием «Background-music module»). `true` — плейлист отдаётся всем посетителям; `false` — только на `127.0.0.1`/`localhost` (локальный предпросмотр). Чтобы поменять плейлист или переключить публичность — правите эти строки и перезапускаете процесс сайта (через операторскую панель или `Start-Sovet-Online.exe`); отдельного конфиг-файла или env-переменной для этого нет.

При экспорте портативной сборки (`server/Export-Portable-Project.ps1`) папка `media\music` копируется только с флагом `-IncludePrivateMusic` — без него на выходе музыки не будет.
